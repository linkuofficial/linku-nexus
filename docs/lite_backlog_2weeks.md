# Nodus Lite Backlog (2 Weeks)

## Week 1

1. Homepage value clarity
- Rewrite headline/subheadline for 5-second understanding.
- Add two clear actions: start exploring / search directly.

2. Search UX improvement
- Tune result ordering by relevance and learning utility.
- Add empty-state guidance with suggested keywords.
- Reduce click steps from result to node detail.

3. Graph readability
- Improve node detail panel readability and hierarchy.
- Ensure labels and controls are legible on mobile width.

## Week 2

1. Learning path conversion
- Add one-click action from node detail to learning path.
- Show progress and next-step hint in learning mode.

2. Content update cadence
- Define weekly batch update checklist.
- Keep naming/tagging conventions consistent.

3. Lightweight release routine
- Run pytest and npm build before release.
- Run post-release smoke: load/search/language toggle.

## KPI Targets

1. New visitors complete a valid search within 30 seconds.
2. Search-result to node-detail click-through increases.
3. Node-detail to learning-path start rate increases.
