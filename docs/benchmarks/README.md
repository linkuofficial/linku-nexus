# Nodus Benchmarks

This folder stores API load benchmark snapshots generated by:

- scripts/load_test_api.py

Conventions:
- `latest.json` / `latest.md`: most recent benchmark in the current branch.
- `latest_guard.json`: most recent guardrail result (pass/fail, violations).
- `nightly_<run_id>.json` / `nightly_<run_id>.md`: CI nightly snapshots.
- `benchmark_YYYYMMDD_HHMMSS.*`: ad-hoc local runs.
- `baseline.json`: baseline reference for regression checks.
- `baseline_history.jsonl`: baseline promotion history with reason + timestamp.
- `drill_guard_fail.json`: optional failure-drill output for notification flow verification.

Example local command:

python scripts/load_test_api.py --base-url http://127.0.0.1:8000 --requests 40 --concurrency 8 --output-dir docs/benchmarks

Guardrail command:

python scripts/benchmark_guard.py --current docs/benchmarks/latest.json --baseline docs/benchmarks/baseline.json --max-regression-pct 35 --max-graph-p95-ms 300 --max-search-p95-ms 120

Baseline promotion command:

python scripts/update_benchmark_baseline.py --reason "post-release calibration"
